4 # Introduction
5 
6 Documentation describes high level python interface to control system of Velma robot.
7 
8 The robot can be controlled using class defined in velma_interface.py.
9 
10 Motion planning is done through planner python interface defined in planner.py.
11 
12 Integration tests are presented in module @ref integration_tests and they can be used:
13  * to evaluate performance and correctness of the whole system,
14  * as tutorials on the interfaces.
15 
16 Additionally, @ref utilities provide useful scripts and ROS launch files.
17 
18 Brief descriptions of some catkin packages are in [Related pages](pages.html) tab.
19 It contains sources for ROS launch files in each package. Please refer to launch files sources to
20 get familiar with setting parameters and running the system and various useful tools.
21 
22 The tab [Modules](modules.html) groups python executable scripts into @ref integration_tests and @ref utilities.

28 # Installation
29 Please follow the steps described in section [Set up: Velma (latest version)](https://github.com/RCPRG-ros-pkg/RCPRG_rosinstall/wiki/Set-up:-Velma-(latest-version)).
30 The simulator is provided in both versions: `latest` and `latest_hw`, but if You don't want to run real hardware, install the simulation-only version (`latest`).

32 # Running the simulator

42  * each subsystem individually (on a single machine):
43  - RE of simulated Velma robot:

51 
52  Please refer to documentation of @ref velma_core_ve_body_readme for details about launch file arguments.

54 
55  roslaunch velma_core_cs velma_core_cs.launch
56 
57  Please refer to documentation of @ref velma_core_cs_readme for details about launch file arguments.

69 ## Running the real HW system:
70 1. Turn on main power supply. It is located in P109 near the main door.
71 
72 2. Turn on lwr-controller PC loctated in server room.
73 
74 3. Turn on both KUKA hardware controllers (in the server room, below lwr-controller PC) and wait till they are fully initialized.
75 
76 4. On both KCP (KUKA Control Panel):
77 
78  1. Click Ackn. All.
79 
80  2. From menu Configure choose 6 set tool/base, set tool no. parameter to 2 and click OK.
81 
82  3. In the file browser select file fri_start and choose Select.
83 
84  4. Switch the Mode Selector key to position Automatic (upper-left).
85 
86  5. Click button Enable drives (the button next to the Mode Selector key).
87 
88  6. Run the script by pushing several times the Start Key button (the green button with + sign). All three indicators at the bottom of the screen should be green.
89 
90 5. The following commands should be executed in multiple terminals. In each terminal, workspace configuration file should be sourced:
91 
92  source path_to_workspace/top/devel/setup.bash
93 
94  and ROS master URI should be set to lwr-controller:
95 
96  export ROS_MASTER_URI=http://lwr-controller:11311
97 
98 6. Execute the following commands on lwr-controller using multiple SSH connections:
99 
100  1. Initialize ROS:
101 
102  roscore
103 
104  2. Run real effector of torso, neck and grippers (devices connected to Ethercat):
105 
106  roslaunch velma_ec_driver velma_ec_re.launch
107 
108  3. Run real effector of right manipulator:
109 
110  roslaunch velma_core_re_lwr velma_re_lwr_r.launch
111 
112  4. Run real effector of left manipulator:
113 
114  roslaunch velma_core_re_lwr velma_re_lwr_l.launch
115 
116  5. Run virtual effector of whole body:
117 
118  roslaunch velma_core_ve_body velma_gazebo_ve_body.launch cset:=true
119 
120  the cset roslaunch argument indicates that the subsystem should run on single, predefined cpu.
121 
122  6. Run control subsystem of the core agent:
123 
124  roslaunch velma_core_cs velma_core_cs.launch cset:=true
125 
126  the cset roslaunch argument indicates that the subsystem should run on single, predefined cpu.
127 
128  7. Run ROS interface of the control subsystem of the task agent:
129 
130  roslaunch velma_task_cs_ros_interface velma_task_cs_ros_interface.launch
