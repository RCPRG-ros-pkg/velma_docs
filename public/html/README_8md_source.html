4 # Introduction
5 
6 Documentation describes high level python interface to control system of Velma robot.
7 
8 The robot can be controlled using class defined in velma_interface.py.
9 
10 Motion planning is done through planner python interface defined in planner.py.
11 
12 Integration tests are presented in module @ref integration_tests and they can be used:
13  * to evaluate performance and correctness of the whole system,
14  * as tutorials on the interfaces.
15 
16 Additionally, @ref utilities provide useful scripts and ROS launch files.
17 
18 Brief descriptions of some catkin packages are in Related pages tab.
19 Modules tab groups python executable scripts into @ref integration_tests and @ref utilities.

26 # Installation
27 Please follow the steps described in section [Set up: Velma (latest version)](https://github.com/RCPRG-ros-pkg/RCPRG_rosinstall/wiki/Set-up:-Velma-(latest-version)).
28 The simulator is provided in both versions: `latest` and `latest_hw`, but if You don't want to run real hardware, install the simulation-only version.

30 # Running the simulator
31 The setup.bash script of the workspace must be sourced in each terminal.
32 Usually is is *setup.bash* script in devel ot install space of top level workspace.
33 
34 Run:
35 1. ROS master:
36 
37  roscore
38 
39 2. Simulated system, two ways are possible:
40  * each subsystem individually (on single machine):
41  - RE of simulated Velma robot:
42 
43  roslaunch velma_sim_gazebo velma_gazebo_re.launch
44 
45  - VE of simulated Velma robot:
46 
47  roslaunch velma_core_ve_body velma_gazebo_ve_body.launch
48 
49  - CS of core agent:
50 
51  roslaunch velma_core_cs velma_core_cs.launch
52 
53  - CS of task agent:
54 
55  roslaunch velma_task_cs_ros_interface velma_task_cs_ros_interface.launch
56 
57  * or all subsystems together:
58 
59  roslaunch velma_common velma_system.launch
60 
61 You can also run:
62 * rqt utility to see the state of the subsystems:
63 
64  rosrun rqt_agent rqt_agent
65 
66 * rviz to see the state of the robot:
67 
68  rosrun rviz rviz
69 
70  You have to add RobotModel and some relevant ROS topics to see the state of the robot.
