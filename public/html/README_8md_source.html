4 # Introduction
5 
6 Documentation describes high level python interface to control system of Velma robot.
7 
8 The robot can be controlled using class defined in velma_interface.py.
9 
10 Motion planning is done through planner python interface defined in rcprg_planner.py.
11 
12 Integration tests are presented in module @ref integration_tests and they can be used:
13  * to evaluate performance and correctness of the whole system,
14  * as tutorials on the interfaces.
15 
16 Additionally, @ref utilities provide useful scripts and ROS launch files.
17

21 # Running the simulator
22 The setup.bash script of the top level workspace must be sourced in each terminal.
23 Usually *setup.bash* script is in devel or install space of top level workspace.

25 ## Running the simulated system:
26 1. Initialize ROS:
27 
28  roscore
29 
30 2. Run subsystems of *velma_core* and *velma_task* agents - two ways are possible:
31  * each subsystem individually (on a single machine):
32  - RE of simulated Velma robot:
33 
34  roslaunch velma_sim_gazebo velma_gazebo_re.launch
35 
36  Please refer to documentation of @ref velma_sim_gazebo_readme for details about launch file arguments.

38 
39  roslaunch velma_core_ve_body velma_core_ve_body.launch
40 
41  Please refer to documentation of @ref velma_core_ve_body_readme for details about launch file arguments.
42  - CS of core agent:
43 
44  roslaunch velma_core_cs velma_core_cs.launch
45 
46  Please refer to documentation of @ref velma_core_cs_readme for details about launch file arguments.
47  - CS of task agent:
48 
49  roslaunch velma_task_cs_ros_interface velma_task_cs_ros_interface.launch
50 
51  Please refer to documentation of @ref velma_task_cs_ros_interface_readme for details about launch file arguments.

53

58 ## Running the real HW system:
59 1. Turn on main power supply. It is located in P109 near the main door.
60 
61 2. Turn on lwr-controller PC loctated in server room.
62 
63 3. Turn on both KUKA hardware controllers (in the server room, below lwr-controller PC) and wait till they are fully initialized.
64 
65 4. On both KCP (KUKA Control Panel):
66 
67  1. Click Ackn. All.
68 
69  2. From menu Configure choose 6 set tool/base, set tool no. parameter to 2 and click OK.
70 
71  3. In the file browser select file fri_start and choose Select.
72 
73  4. Switch the Mode Selector key to position Automatic (upper-left).
74 
75  5. Click button Enable drives (the button next to the Mode Selector key).
76 
77  6. Run the script by pushing several times the Start Key button (the green button with + sign). All three indicators at the bottom of the screen should be green.
78 
79 5. The following commands should be executed in multiple terminals. In each terminal, workspace configuration file should be sourced:
80 
81  source path_to_workspace/top/devel/setup.bash
82 
83  and ROS master URI should be set to lwr-controller:
84 
85  export ROS_MASTER_URI=http://lwr-controller:11311
86 
87 6. Execute the following commands on lwr-controller using multiple SSH connections:
88 
89  1. Initialize ROS:
90 
91  roscore
92 
93  2. Run real effector of torso, neck and grippers (devices connected to Ethercat):
94 
95  roslaunch velma_ec_driver velma_ec_re.launch
96 
97  3. Run real effector of right manipulator:
98 
99  roslaunch velma_core_re_lwr velma_re_lwr_r.launch
100 
101  4. Run real effector of left manipulator:
102 
103  roslaunch velma_core_re_lwr velma_re_lwr_l.launch
104 
105  5. Run virtual effector of whole body:
106 
107  roslaunch velma_core_ve_body velma_core_ve_body.launch cset:=true
108 
109  the cset roslaunch argument indicates that the subsystem should run on single, predefined cpu.
110 
111  6. Run control subsystem of the core agent:
112 
113  roslaunch velma_core_cs velma_core_cs.launch cset:=true
114 
115  the cset roslaunch argument indicates that the subsystem should run on single, predefined cpu.
116 
117  7. Run ROS interface of the control subsystem of the task agent:
118 
119  roslaunch velma_task_cs_ros_interface velma_task_cs_ros_interface.launch
